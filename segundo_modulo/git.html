Anotações sobre Git

Comandos básicos - terminal bash

  • git init → Inicia um repositório Git
  • git status . → Mostra os arquivos
  • git add . → Adiciona todos os arquivos
  • git commit -m "mensagem" → Salva as alterações
  • git log → Mostra o histórico de registros de commits

Trabalhando com repositórios remotos

  • git remote add <nome> <url> → Adiciona um repositório remoto (ex: GitHub)
  • git push <nome> → Envia os commits locais para o repositório remoto (ex: git push origin main)

Permissões e Segurança

Token de acesso:
Um token substitui a senha no GitHub para permitir ações como push e pull com permissões limitadas e prazo de validade.
Você pode gerar tokens no GitHub em:
Settings > Developer Settings > Personal access tokens

.gitignore

O arquivo .gitignore serve para listar arquivos e pastas que o Git deve ignorar.

Git ID (Hash)

Cada commit (e objeto no Git) possui uma ID única de 40 caracteres:
Um hash SHA-1, que serve como referência criptográfica para garantir integridade.

Exemplo de hash: a47c84b1ad7e9f3058439218b2d334c38bbfdd71

Tipos de objetos

  • Blob → conteúdo, sem nome (geralmente em binários)
  • Tree → árvore, faz referência a outras árvores ou blobs
  • Commit → snapshot do conteúdo com metadados
  • TAG → (marcação, exemplo: versão v1.0)

Estados do arquivo

  • Untracked → arquivo novo, ainda não rastreado
  • Unmodified → rastreado, sem modificações
  • Modified → modificado
  • Staged → pronto para commit

Branches e HEAD

  • Branches são linhas paralelas de desenvolvimento independentes
  • Branch padrão: main ou master
  • git branch nome-da-branch → cria nova branch
  • git checkout nome-da-branch → muda para outra branch
  • git switch nome-da-branch → forma moderna de trocar
  • HEAD é o ponteiro para o commit atual (geralmente o último da branch)

Remotes: repositórios Git externos, como GitHub, GitLab, etc. Nome padrão: origin

Comandos úteis:

  • git remote -v → lista repositórios remotos
  • git remote add nome URL → adiciona remoto (ex: git remote add origin https://github.com/usuario/repositorio.git)
  • git push origin nome-da-branch → envia commits
  • git pull origin nome-da-branch → busca e integra alterações
  • git clone URL → clona o repositório

Desfazendo alterações no Git - terminal bash

  • git restore nome-do-arquivo → descarta alterações locais
  • git checkout -- nome-do-arquivo → (antigo) descarta alterações locais
  • git reset nome-do-arquivo → remove da staging sem apagar o conteúdo
  • git reset --soft HEAD~1 → desfaz último commit (mantém staging)
  • git reset --mixed HEAD~1 → desfaz commit e staging (mantém arquivos)
  • git reset --hard HEAD~1 → apaga tudo (commit, staging e arquivos)
  • git revert id-do-commit → cria um commit que desfaz outro (mantém histórico)
  • git push origin nome-da-branch --force → força envio da branch remota
  • git branch -d nome-da-branch → deleta branch local (se já foi mergeada)
  • git branch -D nome-da-branch → força exclusão da branch (sem merge)

Git Stash

Comando para guardar temporariamente alterações não commitadas, permitindo mudar de branch ou realizar outras tarefas sem perder progresso.

Comandos básicos:

  • git stash → guarda todas as alterações não commitadas
  • git stash save "mensagem" → salva com descrição
  • git stash list → mostra a lista de stashes salvos
  • git stash apply → aplica a última stash (sem remover)
  • git stash apply stash@{n} → aplica stash específica
  • git stash pop → aplica e remove do stash
  • git stash drop stash@{n} → remove stash específica
  • git stash clear → remove todas as stashes
